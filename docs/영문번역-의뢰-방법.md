# EN 영문 버전 – 번역 의뢰 방법

## 1. 추천 흐름

1. **노출 한글 수집**  
   `messages/ko.json`에 있는 문구가 곧 “노출되는 한글”입니다. (일부는 이미 영문이 있을 수 있음.)
2. **번역용 파일 만들기**  
   한글만 추려서 **키(key) + 한글(ko) + 영문(en 빈칸)** 형태로 정리해 의뢰처에 넘깁니다.
3. **영문 받은 뒤 반영**  
   의뢰처가 채운 **en** 열을 `messages/en.json` 구조에 맞게 반영합니다.

---

## 2. 한글만 추려서 CSV 만들기 (의뢰용)

프로젝트 루트에서:

```bash
node scripts/export-ko-for-translation.js
```

- **입력:** `messages/ko.json`
- **출력:** `messages/ko-for-translation.csv`  
  - 컬럼: `key`, `ko`, `en`  
  - 한글이 포함된 항목만 포함  
  - `en`은 비워 둔 상태

엑셀에서 열어서 의뢰처에 전달하고, **en 열만 채워서** 돌려받으면 됩니다.

---

## 3. 의뢰 시 전달하면 좋은 것

- **파일:** `ko-for-translation.csv` (또는 동일 구조의 엑셀)
- **안내 문구 예시:**
  - “key는 그대로 두고, **en** 열에만 영문 번역을 채워 주세요.”
  - “화면에 그대로 노출되는 문구이므로, 브랜드/제품 톤에 맞는 영문으로 부탁드립니다.”
- **참고:** 필요하면 `key`가 어디에 쓰이는지(예: 메뉴, 버튼, 본문)만 짧게 표기해 주면 의뢰처가 맥락을 잡기 쉽습니다.

---

## 4. 코드에 남아 있는 한글 (선택)

`messages/ko.json`이 아닌 **페이지/컴포넌트 안에 직접 적힌 한글**이 있을 수 있습니다.  
그런 문구도 영문 버전에 넣으려면:

1. 해당 문구를 `messages/ko.json`에 키와 함께 추가하고
2. 코드에서는 `t("키이름")` 형태로 불러 쓰게 수정한 뒤
3. 위 CSV 추출 스크립트를 다시 돌려 **새로 추가된 키**도 `ko-for-translation.csv`에 포함시키고
4. 그 키에 대해서도 의뢰처에 영문(en)을 요청하면 됩니다.

이렇게 하면 “노출되는 한글만 추려서” 키와 1:1로 매칭해 의뢰할 수 있습니다.
